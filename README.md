# Life Debt（命债）

一个 Fabric 模组：**用未来的生命透支现在的不死**。吃下不死图腾获得「命债」增益——期间可以抵抗致命伤害、越死越难死，但每一次被你赖掉的死亡，都会在增益结束时以**永久生命上限**偿还。还不起，就当场燃尽。

> 本模组是对 [rh666666/Immortality](https://github.com/rh666666/Immortality)（原协议 CC0）功能的重写与多版本移植，重命名为更贴合机制的 **Life Debt**。

## 核心机制

- **不死图腾可食用**：不死图腾变为随时可吃的食物（饱食度 6、饱和 1.0），吃下获得 **5 分钟「命债」增益**。
- **抵抗死亡**：拥有「命债」期间受到致命伤害时不会死亡，而是触发一次与不死图腾等效的复活——补满血、清除所有减益，并获得 再生 II(45s)、伤害吸收 II(5s)、抗火(40s)、夜视(3min)，以及**力量**（在原有等级上 +2，持续 1min）。每次抵抗死亡，死亡计数 +1。
- **越死越难死（叠乘减伤）**：减伤比例
  `R = 1 − 1 / (1 + k · D / H_max)`
  其中 `D` 为本次增益期间累计死亡次数，`H_max` 为当前最大生命值，`k` 为可配置系数（默认 20）。实际受到伤害 = 原伤害 × (1 − R)，并获得等比例的**击退抗性**（上限 100%）。
- **偿还生命上限**：增益结束时，按死亡次数**永久扣除等量的生命上限**。
- **燃尽**：若扣除后生命上限 ≤ 0，玩家以自定义「燃尽」伤害类型死亡（死亡信息「%s 燃尽了」），并获得进度**「燃尽此身」**。
- **复活清偿**：玩家重生后，生命上限惩罚被清除。

死亡计数与会话状态通过玩家 NBT 持久化。

## 配置

配置文件：`config/lifedebt.yml`（首次运行自动生成，含公式注释）。

| 键 | 默认 | 说明 |
|---|---|---|
| `damageReductionK` | `20.0` | 减伤系数 `k`。反解公式 `k = H_max · R / (D · (1 − R))`。例如希望 `H_max=20, D=10` 时 `R=90%`，则 `k=18`。 |

## 支持的版本

单一源码，通过 [Stonecutter](https://stonecutter.kikugie.dev/) 为每个 Minecraft 版本各产出一个 jar：

`1.14.4` · `1.16.5` · `1.17.1` · `1.18.2` · `1.19.2` · `1.19.4` · `1.20.1` · `1.20.4` · `1.20.6` · `1.21.1` · `1.21.4` · `1.21.8`

> 26.x 因改动较大，暂缓，后续再评估。

加载器：Fabric Loader `>=0.19.3`（全版本通用）。需要 Fabric API。

## 构建

需要 JDK（构建时会按各版本自动配置 Java 8/16/17/21 工具链）。构建使用 **Gradle 8.14.4**（通过 wrapper 提供，无需手动安装）。

```bash
# 构建当前激活版本（默认 1.20.1）
./gradlew build

# 切换激活版本
./gradlew "Set active project to 1.21.1"

# 构建全部版本，产物汇总到 build/libs/<mod_version>/
./gradlew chiseledBuild
```
